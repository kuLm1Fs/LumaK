# 7 天执行计划

## 总目标

7 天内做出一个可运行、可展示、可讲解、可测试的“代码库理解与安全修改 Agent”。

## 第 1 天：搭骨架

完成目标：

- 建 repo
- 定目录结构
- 做最小 CLI
- 接通一个 LLM
- 完成一次普通对话

今天的交付标准：

- 可以运行
- README 有最小启动说明

## 第 2 天：做工具调用闭环

完成目标：

- `list_files`
- `read_file`
- `search_text`
- 最小 agent loop

今天的交付标准：

- 用户提问后，模型能决定调用工具
- 工具结果能回喂模型
- 最终回答不是原始工具输出拼接

## 第 3 天：补工程边界

完成目标：

- 参数校验
- 工作目录限制
- 超时控制
- 统一错误提示
- 基础日志

今天的交付标准：

- 工具出错不会直接崩
- 越界路径会被拦截
- 可以追踪每次工具调用

## 第 4 天：补状态与轨迹

完成目标：

- 会话历史保存
- 工具调用轨迹保存
- 明确区分 user / assistant / tool 消息

今天的交付标准：

- 同一个 session 能连续问两轮
- 能看到完整执行轨迹

## 第 5 天：做安全修改能力

完成目标：

- `safe_edit`
- 最好有 diff 预览
- 失败时明确提示

今天的交付标准：

- 可以安全修改示例文件
- 不能越界写入
- 失败时不会静默乱写

## 第 6 天：补测试与评估

完成目标：

- 工具层单测
- 至少 2 个 agent flow 测试
- 准备 10 到 20 个手工评测任务
- 记录成功与失败样例

今天的交付标准：

- 不只是“感觉可用”
- 而是“有测试和任务集支撑”

## 第 7 天：做展示材料

完成目标：

- 打磨 README
- 画简易架构图
- 准备执行轨迹示例
- 录一个 1 到 3 分钟 demo
- 写一页设计说明

今天的交付标准：

- 别人 clone 后能跑
- 你能讲 10 到 15 分钟
- GitHub 页面看起来不是玩具

## 最低成功线

只要满足下面 6 条，这个项目就已经具备面试展示价值：

1. 能跑
2. 有 3 到 4 个工具
3. 有 agent loop
4. 有日志或执行轨迹
5. 有测试
6. 有 README 和 demo
