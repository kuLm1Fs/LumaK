# 项目方向

## 最终方向

做一个“本地代码库理解与安全修改 Agent”。

它不是通用聊天助手，也不是 AI Trello，而是一个面向开发者的 agent，专门解决这类问题：

- 这个项目入口在哪
- 这个模块负责什么
- 帮我找出和某个功能相关的代码
- 总结这个仓库的架构
- 根据要求修改一个文件并说明改动

## 为什么定这个方向

这个方向最适合当前阶段拿来做面试主项目，因为它同时满足四件事：

1. agent 味足够强
   - 模型需要决定何时调用什么工具
   - 需要多步工具调用
   - 不是一次性问答

2. 工程化空间足够大
   - 工具边界
   - 路径限制
   - 日志
   - 状态管理
   - 测试
   - 错误恢复

3. 不容易做成花架子
   - 前端不是重点，runtime 才是重点
   - 面试官更容易理解价值

4. 后续能自然接 TS
   - 先做 Python agent runtime
   - 后面再补一个 TS/React 前端

## 为什么不选别的

### 不选 AI Trello / AI Todolist

问题不是完全不能做，而是当前版本太容易变成“CRUD + LLM 排序器”。

它的问题在于：

- agent 味不够纯，更像智能推荐功能
- UI 工作量大，容易稀释核心能力
- 优先级排序强依赖上下文，评估难且容易幻觉
- 很容易把时间花在看板交互而不是 agent runtime 上

### 不选大而全的通用 Agent 平台

问题在于：

- 容易堆功能
- 容易失控
- 容易每个点都浅
- 很难在短时间内做出像样的工程质量

## 这个项目最终要证明什么

完成后，它应该能证明：

- 会设计 agent loop
- 会做 tool calling
- 会做运行时约束
- 会做最小状态管理
- 会做工程化而不是 prompt 拼接
- 会把一个小项目做出非玩具感
